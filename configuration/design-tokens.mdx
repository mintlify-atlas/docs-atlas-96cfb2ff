title: Design Tokens
description: Customize colors, typography, spacing, and visual style with design tokens
---

Design tokens allow you to customize the visual style of generated Figma assets while maintaining consistency across all diagrams and components. Tokens override the default design system values.

## Configuration

```yaml .omni-architect.yml
design_system: "material-3"  # Base system

design_tokens:
  colors:
    primary: "#4A90D9"
    secondary: "#7B68EE"
    success: "#2ECC71"
    error: "#E74C3C"
    warning: "#FFA500"
  typography:
    font_family: "Inter"
    heading_size: 24
    body_size: 14
  spacing:
    base: 8
    scale: 1.5
```

## Token Categories

### Colors

Define the color palette for all generated assets.

#### Basic Color Tokens

```yaml
design_tokens:
  colors:
    primary: "#4A90D9"      # Primary brand color
    secondary: "#7B68EE"    # Secondary brand color
    success: "#2ECC71"      # Success states
    error: "#E74C3C"        # Error states
    warning: "#FFA500"      # Warning states
```

#### Extended Color Tokens

```yaml
design_tokens:
  colors:
    # Brand Colors
    primary: "#4A90D9"
    secondary: "#7B68EE"
    tertiary: "#1ABC9C"
    
    # Semantic Colors
    success: "#2ECC71"
    error: "#E74C3C"
    warning: "#FFA500"
    info: "#3498DB"
    
    # Neutral Scale (for custom design systems)
    neutral_50: "#F9FAFB"
    neutral_100: "#F3F4F6"
    neutral_200: "#E5E7EB"
    neutral_300: "#D1D5DB"
    neutral_400: "#9CA3AF"
    neutral_500: "#6B7280"
    neutral_600: "#4B5563"
    neutral_700: "#374151"
    neutral_800: "#1F2937"
    neutral_900: "#111827"
    
    # Surface Colors
    background: "#FFFFFF"
    surface: "#F9FAFB"
    surface_variant: "#E5E7EB"
    
    # Text Colors
    text_primary: "#111827"
    text_secondary: "#6B7280"
    text_disabled: "#9CA3AF"
    text_on_primary: "#FFFFFF"
    
    # Border Colors
    border: "#E5E7EB"
    border_focus: "#4A90D9"
    border_error: "#E74C3C"
```

### Typography

Control font families, sizes, weights, and text styles.

#### Basic Typography Tokens

```yaml
design_tokens:
  typography:
    font_family: "Inter"         # Primary font
    heading_size: 24             # Heading size in px
    body_size: 14                # Body text size in px
```

#### Extended Typography Tokens

```yaml
design_tokens:
  typography:
    # Font Families
    font_family: "Inter"
    font_family_mono: "Fira Code"
    font_family_display: "Playfair Display"
    
    # Type Scale (sizes in px)
    heading_1: 32
    heading_2: 28
    heading_3: 24
    heading_4: 20
    heading_5: 18
    heading_6: 16
    body_large: 16
    body_size: 14               # body_medium
    body_small: 12
    caption: 11
    overline: 10
    
    # Font Weights
    weight_light: 300
    weight_regular: 400
    weight_medium: 500
    weight_semibold: 600
    weight_bold: 700
    weight_extrabold: 800
    
    # Line Heights (unitless multipliers)
    line_height_tight: 1.2
    line_height_normal: 1.5
    line_height_relaxed: 1.75
    
    # Letter Spacing (in em)
    letter_spacing_tight: -0.02
    letter_spacing_normal: 0
    letter_spacing_wide: 0.05
```

### Spacing

Define spacing scale for consistent layouts.

#### Basic Spacing Tokens

```yaml
design_tokens:
  spacing:
    base: 8           # Base unit in px
    scale: 1.5        # Multiplier for scale (8, 12, 18, 27...)
```

#### Extended Spacing Tokens

```yaml
design_tokens:
  spacing:
    base: 8           # Base unit
    scale: 1.5        # Scale factor
    
    # Named spacing (in px or using base multiples)
    xs: 4             # 0.5 × base
    sm: 8             # 1 × base
    md: 16            # 2 × base
    lg: 24            # 3 × base
    xl: 32            # 4 × base
    xxl: 48           # 6 × base
    xxxl: 64          # 8 × base
    
    # Component-specific spacing
    button_padding_x: 16
    button_padding_y: 8
    card_padding: 16
    section_gap: 24
    page_margin: 32
```

#### Spacing Scale Systems

<Tabs>
  <Tab title="Linear (8pt Grid)">
    ```yaml
    spacing:
      base: 8
      scale: 1        # 8, 16, 24, 32, 40, 48...
      # Results in: 8, 16, 24, 32, 40, 48, 56, 64
    ```
  </Tab>
  
  <Tab title="Fibonacci-like">
    ```yaml
    spacing:
      base: 8
      scale: 1.5      # 8, 12, 18, 27, 40, 60...
      # Results in: 8, 12, 18, 27, 40, 60, 90
    ```
  </Tab>
  
  <Tab title="Tailwind (4pt)">
    ```yaml
    spacing:
      base: 4
      scale: 2        # 4, 8, 16, 32, 64...
      # Results in: 4, 8, 16, 32, 64, 128
    ```
  </Tab>
</Tabs>

    modal: 16

## Design System Overrides

Tokens override the base design system values:

### Material 3 Override Example

```yaml
design_system: "material-3"  # Base: Material 3 defaults

design_tokens:
  colors:
    primary: "#FF6B00"  # Override Material purple with brand orange
    # Other Material 3 colors (secondary, error, etc.) remain default
  typography:
    font_family: "Poppins"  # Override Roboto with custom font
    # Material 3 type scale remains
```

### Tailwind Override Example

```yaml
design_system: "tailwind"  # Base: Tailwind defaults

design_tokens:
  colors:
    primary: "#8B5CF6"  # Override blue-500 with violet-500
  spacing:
    base: 4             # Keep Tailwind's 4px base
    scale: 2            # Keep Tailwind's doubling scale
```

### Custom Design System Example

```yaml
design_system: "custom"  # No defaults - all tokens required

design_tokens:
  colors:
    primary: "#4A90D9"
    secondary: "#7B68EE"
    success: "#2ECC71"
    error: "#E74C3C"
    warning: "#FFA500"
    # ... (all color tokens required)
  typography:
    font_family: "Inter"
    heading_size: 24
    body_size: 14
    # ... (all typography tokens recommended)
  spacing:
    base: 8
    scale: 1.5
    # ... (all spacing tokens recommended)
  border_radius:
    sm: 4
    md: 8
    lg: 12
    # ... (optional but recommended)
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Semantic Color Names">
    Define colors by purpose (`primary`, `error`) rather than appearance (`blue`, `red`) to maintain flexibility.
  </Accordion>
  
  <Accordion title="Maintain Consistent Spacing Scale">
    Use a consistent base unit (4px or 8px) and scale factor throughout your design system.
  </Accordion>
  
  <Accordion title="Test Color Contrast">
    Ensure text colors meet WCAG AA standards (4.5:1 for normal text, 3:1 for large text).
  </Accordion>
  
  <Accordion title="Document Custom Tokens">
    When using `design_system: custom`, create comprehensive documentation for all token values and their usage.
  </Accordion>
  
  <Accordion title="Sync with Code Tokens">
    Keep design tokens in sync with your frontend code tokens (CSS variables, Tailwind config, etc.) to reduce handoff friction.
  </Accordion>
  
  <Accordion title="Version Control Tokens">
    Commit `.omni-architect.yml` to version control so design tokens evolve with your codebase.
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Invalid Color Format

**Error**: `Invalid color format for 'primary'`

**Solution**: Use hex (#4A90D9), rgb(), rgba(), hsl(), or hsla() formats:
```yaml
colors:
  primary: "#4A90D9"  # ✓ Correct
  # primary: "blue"  # ✗ Invalid
```

### Font Not Available in Figma

**Warning**: `Font 'CustomFont' not found in Figma, falling back to 'Inter'`

**Solution**: Ensure the font is installed in Figma or use a standard font:
```yaml
typography:
  font_family: "Inter"  # ✓ Available in Figma
  # font_family: "MyCustomFont"  # ✗ Not in Figma
```

### Spacing Scale Not Applied

**Issue**: Components don't follow spacing scale

**Solution**: Verify `base` and `scale` are numbers (not strings):
```yaml
spacing:
  base: 8      # ✓ Correct (number)
  # base: "8"  # ✗ Invalid (string)
  scale: 1.5   # ✓ Correct
```
